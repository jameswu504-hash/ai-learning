# 初學者維護指南

你現在的網站有兩種保存方式：

## 方式一：站內新增內容

在網站打開「我的內容」頁，可以新增：

- 筆記
- 技能
- 資源
- 作品

這些資料會存在瀏覽器的 localStorage。

適合用來練習、暫存、整理想法。

限制：

- 換瀏覽器或換電腦就看不到。
- 清除瀏覽器資料可能會消失。
- 別人打開網站不會看到你的 localStorage 內容。

所以要定期按「產生備份 JSON」，把內容保存起來。

## 方式二：正式更新網站

等你確定內容要變成網站的一部分，再把它寫進 `data.js`。

流程：

1. 在「我的內容」先新增筆記。
2. 整理成熟後匯出 JSON。
3. 把適合公開的內容搬到 `data.js`。
4. 本機打開 `index.html` 檢查。
5. commit。
6. push 到 GitHub。

## 你每週要做的事

1. 新學到的東西先放到「我的內容」。
2. 週末整理一次。
3. 能變成正式知識的內容，搬進 `data.js`。
4. 不確定的內容留在「我的內容」或 inbox。
5. 每月備份一次 JSON。

## 重要觀念

localStorage 是你的草稿本。

`data.js` 是網站正式內容。

GitHub 是網站的歷史紀錄與發布來源。
