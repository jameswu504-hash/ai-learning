一、AI 產業鏈一頁圖

AI 不是只有模型。真正的 AI 系統是一條從物理能源到使用者應用的長鏈，每一層都會影響成本、速度、能力與商業模式。

01能源層

AI 的物理極限：電力、冷卻、穩定供應。

大型資料中心需要巨量電力與散熱，能源成本會直接影響訓練與推理成本。

你要懂：AI 不是純軟體，它受限於電力、散熱與基礎建設。

02晶片層

AI 的發動機：GPU、加速器、CUDA 生態。

CPU 擅長複雜邏輯與循序處理，GPU 擅長平行矩陣運算，因此特別適合深度學習。

你要懂：NVIDIA 的強不只在晶片，也在 CUDA 開發者生態。

03Infra 層

AI 的身體：雲端、資料中心、儲存、網路與萬卡叢集。

這一層負責把大量 GPU、網路、儲存與任務排程整合起來，讓模型能被訓練與部署。

你要懂：真正困難的是把大量算力穩定、低延遲、高效率地組織起來。

04模型層

AI 的大腦：訓練、推理、微調、對齊與評估。

模型先透過大量資料訓練出通用能力，再透過推理服務使用者，也可用微調適配特定任務。

你要懂：訓練是養成能力，推理是實際幹活，微調是讓模型更貼近特定場景。

05應用層

AI 與世界互動的窗口：產品、API、工作流與使用者介面。

應用可以是既有產品加入 AI，也可以是從大模型能力重新設計的 AI-native 產品。

你要懂：Token 與 API 是應用層使用模型能力時最常見的計量與連接方式。

二、Token / Embedding 解釋卡

Token

Token 是模型讀文字的最小工作單位

模型不直接理解人類文字，而是先透過 Tokenizer 把文字切成片段，再轉成數字 ID。英文可能以字、詞根或片段切分；中文可能是一個字、詞或更細的片段。

「我想學 AI」 -> ["我", "想", "學", " AI"] -> [token_id_1, token_id_2, token_id_3, token_id_4]

重點：Token 影響成本、上下文長度、處理速度與多語言效率。

Embedding

Embedding 是把文字放進語意空間

Embedding 會把文字、句子或文件轉成一串向量數字。語意越接近，向量距離通常越近；語意越遠，距離通常越遠。

Embedding("貓") ≈ Embedding("狗")
Embedding("貓") 與 Embedding("發票") 距離較遠

重點：Embedding 是 RAG、語意搜尋、推薦、聚類與相似度比對的基礎。

One-hot 的問題

One-hot 只知道「是不是同一個詞」，不知道語意接近。例如貓和狗在 one-hot 中可能完全垂直，無法表達牠們都屬於動物。

稠密向量的價值

Embedding 用較少維度承載語意關係，能讓模型知道「貓、狗、寵物」比「貓、發票、伺服器」更接近。

多模態的方向

現代模型會把文字、圖片、音訊放進可對齊的表示空間，讓模型能理解不同媒介之間的關係。

三、訓練、推理、微調差異表

概念 它在做什麼 像什麼 成本特性 你要判斷的事
訓練 Training 用大量資料讓模型學會語言、知識、模式與推理能力。 培養一個通才大腦。 極高成本，需要大量 GPU、資料、電力與時間。 通常不是個人或一般公司從零做，除非有巨大資源。
推理 Inference 使用已訓練好的模型回答問題、生成內容或執行任務。 讓大腦實際工作。 每次使用都會產生成本，常以 token、延遲、吞吐量計算。 產品體驗會受速度、穩定性、成本與輸出品質影響。
微調 Fine-tuning 用特定資料讓通用模型更適合某個領域、格式、語氣或任務。 把通才訓練成專科助理。 比從零訓練便宜，但仍需要乾淨資料與評估。 不是所有問題都要微調；很多場景可先用提示詞、RAG 或工具解決。

四、20 個核心名詞表

01AI

讓機器執行需要人類智慧的任務，例如理解、生成、判斷與規劃。

02LLM

大型語言模型，透過大量文字與資料學習語言模式並生成回應。

03Token

模型處理文字的基本單位，可能是字、詞或文字片段。

04Tokenizer

把文字切成 Token 並轉成模型可處理數字 ID 的工具。

05Embedding

把文字、句子、文件轉成向量，用距離表示語意相似度。

06向量資料庫

儲存 Embedding，支援用語意相似度快速找相關內容。

07訓練

用大量資料更新模型參數，讓模型形成通用能力。

08推理

使用訓練好的模型處理輸入並產生輸出。

09微調

用特定資料讓模型更適合特定領域、格式或任務。

10參數

模型內部的權重數值，承載模型學到的模式。

11GPU

擅長平行運算的處理器，適合深度學習中的矩陣計算。

12CUDA

NVIDIA 的 GPU 軟體開發生態，讓開發者能有效使用 GPU 算力。

13資料中心

集中放置伺服器、GPU、儲存與網路設備的基礎設施。

14API

應用程式呼叫模型或服務的介面，是 AI 能力接到產品的橋樑。

15上下文

模型在一次對話或任務中可參考的輸入資訊範圍。

16幻覺

模型生成看似合理但不正確、無來源或編造的內容。

17RAG

檢索增強生成，讓模型先查資料再回答，降低胡說風險。

18Chunking

把文件切成適合檢索的知識片段，不只是機械切字數。

19AI-native

以 AI 能力為核心重新設計業務邏輯與產品介面的產品型態。

20資料飛輪

使用者行為與 bad case 回饋持續改善資料、模型與產品體驗的循環。
